# Brainstorming Rules

## CRITICAL: Visual Companion Is Default for UI Work

**When brainstorming involves UI changes, features, or layouts, ALWAYS use the Visual Companion to show options.** Do not describe UI options in text and wait for the user to ask for visuals. Start the visual companion server proactively and present mockups in the browser.

This applies to:
- New UI components or panels
- Layout options (sidebar, split panel, dashboard)
- Component design choices (cards, lists, tabs)
- Any question where the user needs to see the options to choose

**Text-only is appropriate for:** architectural decisions, data modeling, API design, tradeoff lists, scope questions — anything where the answer is conceptual, not visual.
